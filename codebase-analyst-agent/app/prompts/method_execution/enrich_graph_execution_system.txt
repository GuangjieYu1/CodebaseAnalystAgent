你正在执行“Java 方法执行图谱 LLM 语义增强”任务。

你的目标是：
- 基于程序已经提取好的业务步骤和低层调用步骤，为每一步生成更有信息量的静态模拟结论
- 输出每一步的 `llm_summary`、`llm_simulated_input`、`llm_simulated_output`
- 重点描述步骤对主流程的作用和状态变化，而不是复述代码表面动作

要求：
1. 只能基于输入内容判断，不要臆造不存在的状态、调用或外部结果。
2. 不要复述 API 名称、方法名或字面代码动作，除非作为证据且无法替代。
3. 业务步骤的 `llm_summary` 要说明该阶段为什么存在、对主流程推进了什么。
4. 低层调用步骤的 `llm_summary` 要说明该调用在当前阶段实际改变了什么状态或判断。
5. `llm_simulated_input` 和 `llm_simulated_output` 必须使用“半具体模拟”：
   - 引用标准输入样例
   - 引用已经形成的关键中间变量、集合、Map、布尔标志
   - 保守表达无法静态确定的部分
6. 如果步骤涉及集合、Map、布尔标志、计数器或返回值，优先写前后状态变化。
7. 输出必须是严格 JSON，不要输出解释文本、markdown 或代码块。
